# 🥇 메달리온 아키텍처 (Medallion Architecture)

데이터 레이크하우스(Data Lakehouse) 환경에서 데이터를 논리적으로 정리하고 관리하기 위해 사용하는 데이터 설계 패턴이다. 데이터가 각 레이어(브론즈 $\rightarrow$ 실버 $\rightarrow$ 골드)를 통과할수록 데이터의 구조와 품질이 점진적으로 개선되는 것이 핵심 목표이다. 

여러 단계를 거쳐 데이터가 정제되기 때문에 멀티 홉(Multi-hop) 아키텍처라고도 불린다.

> **멀티 홉(Multi-hop)이란?**   
>  데이터나 신호가 출발지(Source)에서 목적지(Target)로 갈 때, 한 번에 직접 연결되지 않고 하나 이상의 중간 노드(레이어)를 거쳐 전달되고 가공되는 방식

## 🥉 Bronze Layer (브론즈 레이어)

외부의 다양한 소스(DB, API, 로그 파일 등)로부터 가져온 **가공되지 않은 원본 데이터(Raw Data)** 가 적재되는 계층. 


* 원본 데이터의 이력을 완벽하게 보존하는 것이 목적이다. 
* 데이터의 로드 날짜, 시간, 프로세스 ID 등의 메타데이터만 추가된 상태로 저장된다. 
* 데이터 계보(Lineage) 추적과 감사(Audit)을 위한 불변의 기록을 제공한다. 
* 데이터 가공이 필요한 경우, 소스로부터 데이터를 다시 가져오지 않아도 된다. 



## 🥈 Silver Layer (실버 레이어)
브론즈 레이어의 **원본 데이터를 파싱, 정제, 필터링한 데이터**  

* 데이터 사이언티스트나 부서별 데이터 애널리스트가 분석을 시작할 수 있는 신뢰할 수 있는 소스 역할을 한다. 
* 데이터 품질 검사, 중복 제거, 표준화 작업을 거쳐 비즈니스 규칙에 맞게 유효성 검사가 수행된다. 
* 변경 데이터 캡처(CDC) 기술 등을 활용해 변경된 데이터만 증분 처리하여 효율성을 높인다.

> **"엔터프라이즈 뷰(Enterprise View)"** 를 제공한다
> 
> 여기서 `엔터프라이즈(Enterprise)`는 '회사(전사) 전체'를 의미한다. 부서마다 데이터를 다르게 정의하는 것(예: 영업팀의 '고객' 기준과 마케팅팀의 '고객' 기준이 다름)을 방지하고, **회사 전체가 공통으로 이해하고 사용할 수 있도록 표준화된 하나의 통합된 관점(View)** 을 제공한다는 뜻이다. 

## 🥇 Gold Layer (골드 레이어) 
실버 레이어의 데이터를 바탕으로 **특정 비즈니스 목적이나 분석 사용 사례에 맞게 집계되고 가공된 최종 테이블**

* 대시보드 시각화, 비즈니스 인텔리전스(BI) 리포트, 머신러닝 모델 학습, 최종 사용자의 즉각적인 쿼리 실행 등에 **바로 사용할 수 있는 상태**를 제공한다.
* 데이터베이스의 '스타 스키마(Star Schema)' 형태로 구성되는 경우가 많으며, 프로젝트별/부서별로 최적화되어 있다.

### 요약

| 레이어 | 데이터 상태 | 주요 역할 | 주 사용자 |
| :--- | :--- | :--- | :--- |
| **Bronze** | Raw (가공되지 않은 원본) | 데이터 이력 보존, 원천 시스템 부하 방지 | 데이터 엔지니어 |
| **Silver=** | 정제 및 표준화. 유효성 검사. | 전사적 단일 진실 공급원(SSOT) 제공 | 데이터 사이언티스트, 데이터 분석가 |
| **Gold** | 분석 맞춤형 집계 | 비즈니스 레벨의 분석 및 시각화 지원 | 비즈니스 사용자, BI 분석가 |
