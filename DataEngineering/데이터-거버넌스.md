## 데이터 거버넌스 (Data Governance)
데이터가 조직 내에서 어떻게 저장되고 관리되어야 하는지를 정의하는 정책, 표준, 절차의 포괄적인 프레임워크를 가리킨다. 즉, 데이터 소유권, 조직 내 데이터 관리에 대한 역할 및 책임을 정의하고, 데이터 정책 및 절차를 강제하는 과정이다.

데이터 관리란, 데이터가 적절하게 수집, 저장, 유지되고 사용자에게 접근 가능하도록 하는 일상적인 기술적 및 운영적 과정이다. 데이터 관리는 거버넌스 규칙에 따라 데이터가 어떻게 처리되고 사용하는지를 나타내며, 데이터 거버넌스 전략에 따라 데이터 생애 주기를 기술적으로 구현되는 것이다.

정리하면, 데이터 관리가 집을 지을 도구라면, 데이터 거버넌스는 청사진이다.
데이터 관리와 데이터 거버넌스는 헷갈릴 수 있지만, 분명히 다른 목적을 가지고 있다.

이론적으로는 데이터 거버넌스가 선행되어 데이터 관리가 되어야할 것 같지만, 실무에서 위를 준수하기는 쉽지 않다.  
우리 회사의 경우, 규칙을 만들지 않은 채로 ETL 파이프라인을 구축하고 데이터를 적재하여 이 데이터의 원천이 어디인지 사용되고 있는 건지를 알기가 매우 어렵다. 따라서 최근 hadoop을 doris로 전환하면서 어떤 데이터가 유의미한지를 파악하기가 어려워 애를 먹고 있다. 뒤늦게 openmetadata를 도입하여 데이터 거버넌스를 만드려고 하는 중이다.

### 핵심 요소
1. Discoverability & Cataloging: 이 데이터를 어디서 찾아?
2. Lineage: 이 데이터 어떻게 만들어진거야? (계보)
3. Data Quality: 이 데이터 믿어도 돼? (규칙)
4. Security & Privacy: 이 데이터 아무나 봐도 돼? (권한)
